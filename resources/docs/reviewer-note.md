# Note to Reviewers

Thank you for reviewing this draft. It is being shared with volunteer reviewers
from the Datalevin community before publication.

You do not need to review the whole book. Feedback on one chapter, one section,
or one example is useful.

I would especially appreciate comments on:

- technical correctness;
- clarity of explanations;
- missing topics or confusing parts;
- code examples that need improvement;
- places where Datalevin behavior, API coverage, or production guidance needs
  sharper qualification.

When sending feedback, please include the chapter number, section heading, and a
short quote or code fragment when possible. That makes it much easier to locate
and fix the issue.

The draft still may change before publication. Formatting, diagrams, examples,
chapter order, and PDF layout should be treated as provisional.
